  1. 논리
    • Hard vs. Soft Logic
      • 카드 문제

        사실: 모든 카드의 한쪽에는 알파벳이, 다른 쪽에는 숫자가 써 있음

        주장: 만약 한쪽이 D 이면 반대쪽은 3

        주장이 사실인지 확인하기 위해 다음 카드들 중 반드시 뒤집어 보아야 하는 것은 몇 개이고 어느 것인가?

        카드: |D| |F| |3| |7|

        생각한 답: 1 개만 뒤집어 보면 됨. |D| or |3| 카드

        답: 2 개를 뒤집어 봐야 함. |D| and |7| 카드

        풀이

        • |D| 카드를 뒤집어 보아야 한다는 것은 누구나 알아 냄
        • |F| 카드를 뒤집어 봤을 때 어떤 숫자가 있든 주장이 사실인지 여부에 영향이 없음 숫자가 3 이어도 주장과 상관이 없고, 3 이외의 다른 숫자여도 주장과 상관이 없음
        • |3| 카드를 뒤집어 봤을 때 어떤 영어가 있든 주장이 사실인지 여부에 영향이 없음 주장에서 정해진 것은 D 일 때 3 이기 때문에, 영어가 F여도 주장과 상관이 없음
        • |7| 카드를 뒤집어 봤을 때 D가 있다면 주장이 성립하지 않음 주장에서 정해진 것은 D 일 때 3 이기 때문임

      • 맥주집 문제

        규칙: 20 세 이하인 사람은 맥주를 마실 수 없음

        문제: 나이 혹은 마시고 있는 것을 표시한 다음 4 명 중 확인이 필요한 사람은 몇 명이고 누구인가?

        사람: |17세| |31세| |콜라| |맥주|

        생각한 답: 2 명. |17세| and |맥주|

        답: 2 명. |17세| and |맥주|

      • 카드 문제와 맥주집 문제의 비교

        카드 문제와 맥주집 문제는 완전히 같은 문제임 (논리적 구성은 완전히 동일함)

        카드 문제를 풀 때는 논리를 이용했지만, 맥주집 문제를 풀 때는 논리를 이용해서 풀지 않고 직관을 사용함

      • 직관

        논리적인 느낌을 주는 것

        어렸을 때 논리적인 문제를 많이 풀었기 때문에, 그 경험으로 논리를 이용하지 않고 직관을 사용해도 문제를 풀 수 있게 된 것

        장점: 익숙한 상황일 경우 빠름

        단점: 강한 착각을 일으킬 수 있어 정확하지 않을 수 있음

      • 진짜 논리 vs. 직관적인 논리
        1. 너 과자 몇 개 먹었니? vs. 버스 타려고 하는데 천원 있니?

          1 번 질문의 경우 과자 3 개를 먹었어도 1 개를 먹었다고 대답은 할 수 있음 (1 개씩 3번 먹었다는 것은 사실임) 하지만 일반적으로 1 개를 먹었다고 하면 이해하기 어렵고, 먹은 개수를 다 말해줌

          2 번 질문의 경우 만원을 가지고 있어도 천원을 가지고 있다고 대답함 (만원에 천원이 포함되어 있기 때문임) 만원을 가지고 있었는데 왜 천원이 있다고 대답했냐고 역으로 질문해도 질문에 대한 답은 정확하게 주었기 때문에 틀린 내용이 없음

          두 질문은 같은 표현을 사용하지만, 하나는 정확한 개수를 요구하고 다른 하나는 천원 이상이 있는지 물어보는 것 1 번 질문은 진짜 논리, 2 번 질문은 직관적인 논리

        1. 합격하려면 토플 500 점 이상 혹은 토익 600 점 이상이 필요 vs. 복권에 당첨되면 자동차 혹은 천만원을 줍니다

          1 번 질문의 경우 토플 500 점 이상과 토익 600 점 이상 모두 취득하고 있어도 합격할 수 있음

          2 번 질문의 경우 복권에 당첨되어도 자동차와 천만원을 모두 받을 수 없음

          두 말은 같은 표현을 사용하지만 하나는 inclusive or, 다른 하나는 exclusive or

        이러한 차이가 직관으로 들어와 있기 때문에, 일상에서 아무 문제없이 위의 표현을 사용함

        하지만 익숙하지 않은 프로그래밍에서 이러한 상황이 등장하게 되면 심한 착각을 불러일으킬 수 있음 (논리를 따로 연구해야할 필요가 있음)

      • 일상 생활 vs. 프로그래밍

        일상 생활에서는 Soft Logic 이 빠르기 때문에 유용함 논리적으로 부정확한 표현을 사용하지만, 어떤 의미인지 모든 사람이 알고있기 때문임

        프로그래밍에서는 Hard Logic 을 사용함 프로그래밍 언어의 표현들이 모두 논리학에서 나왔고, 사용되는 수많은 알고리즘을 이해하기 위해서는 Hard Logic 이 필요함 이러한 이유로 직관을 사용해 프로그래밍을 하려고 하면 어렵게 느껴지게 됨

      • 오해의 근원

        Soft Logic 으로 알고리즘을 이해하려고 해서 프로그래밍이 어려운 것

        알고리즘 설명을 보고 또 봐도 이해가 안 되는 것은 증명을 안 봤기 때문임

      문제 1: 다음을 명제식 형태로 쓰고 참인지 거짓인지 판단하시오

      1. 만약 0 이 홀수라면, 미국에서 2080 년 월드컵이 열린다.

      생각한 답:

      답: 참

      풀이

      • 미국에서 2080 년에 월드컵이 열리는지에 대한 내용은 아무도 모르지만, 전체 명제식이 사실인지 아닌지 확인할 수 있는 방법이 있음
      • 0 이 홀수가 아니라는 것은 알고있음 (p 가 거짓인 것을 알 수 있음)
      • 가정이 거짓이면 전체 명제식은 항상 참이 됨
      • p → q 라는 명제식에서, ~p 인 경우 전체 명제식은 참이 됨 *1
      • 시험에서 100 점을 받으면, 치킨을 사준다 라는 명제가 있다고 가정 시험에서 100 점을 받았는데 치킨을 사줬는지에 대한 여부는 명확히 나타낼 수 있지만, 시험에서 100 점을 받지 못했을 경우는 치킨을 사줬는지에 대한 여부를 따질 수 없기 때문에 해당 명제는 무조건 참이 됨

      1. 만약 19893827938274839 이 Prime Number 라면, 2 는 짝수이다.

      생각한 답: 참

      답: 참

      풀이

      • 19893827938274839 라는 숫자가 Prime Number 인지 판별할 수 없음
      • 2 가 짝수라는 것은 알고있음 (q 가 참인 것을 알 수 있음)
      • p → q 라는 명제식에서, q 가 참인 경우 전체 명제식은 항상 참이 됨
      • p → q 라는 명제식의 대우 명제는 ~q → ~p 인데, ~q 가 거짓이기 때문에 위의 예시에서 볼 수 있듯이 전체 명제식이 참이 됨
      • 대우 명제식이 참일 경우, 본 명제식도 참이 됨

      문제 2: p 와 q 가 명제이고 p → q 가 거짓이라고 하자. 다음 명제식의 참 거짓은 어떻게 되는가?

      1. ~p → q
      1. p v q
      1. q → p

      생각한 답: 1. 참 2. 참 3. 참

      답: 1. 참 2. 참 3. 참

      풀이

      • p → q 가 거짓인 경우는 단 한 가지 밖에 없음 (p 가 참이고, q 가 거짓인 경우)
      • 1 번의 경우 ~p 는 거짓이므로 전체 명제식은 참이 됨
      • 2 번의 경우 or 연산은 두 개가 모두 거짓일 때만 거짓이기 때문에 참이 됨
      • 3 번의 경우 q 가 거짓이므로 전체 명제식은 참이 됨

  1. 증명
    • 글로만 쓰는 것이 아닌 정확한 명제식으로 표현할 수 있는 것이어야 함 증명에 대한 수많은 오해가 p → q 를 p ↔ q 와 혼동하는 것에서 일어남
    • 당구공 Paradox

      문제: 모든 당구공은 색이 같다는 다음 증명에서 잘못된 것은?

      • 수학적 귀납법: P(1) 이 참이고, P(n) → P(n + 1) 이 참이면 P(n) 은 모든 자연수 n 에 대해서 참이다. 어떤 명제 P(n) 이 참이라는 것을 증명하기 위해서 P(1) 이 참이라는 것과 P(n) 이 참이면 P(n + 1) 이 참이라는 것을 증명하면 모든 자연수 n 에 대해서 P(n) 이 참이라는 것을 증명할 수 있음
      • 모든 자연수 n 에 대해 당구공 n 개가 들어있는 집합에서 그 집합에 포함된 당구공은 모두 색이 같다는 것을 증명함 n 개 짜리 집합을 만들면, 어떠한 n 개 짜리 집합이든 색이 같다는 것을 증명하고자 함 어떤 집합이라도 당구공이 n 개 포함된 집합이면, 그 집합에 포함된 당구공은 모두 색이 같다는 것을 증명함
      • P(1): 당구공 1 개가 들어있는 집합은 모두 색이 같음
      • P(n) → P(n + 1) 을 증명하기 위해 P(n) 이 참이라고 가정함
      • 당구공 n + 1 개가 들어있는 임의의 집합을 생각함
      • 이 집합에서 하나를 빼면 당구공 n 개가 있는 집합이 되므로 지금 상황에서 모든 당구공의 색이 같음 P(n) 이 참이라고 가정했기 때문임
      • 방금 뺀 원소를 다시 넣고, 다른 당구공을 빼면 역시 당구공 n 개가 있는 집합이 되므로 지금 상황에서도 모든 당구공의 색이 같음
      • 위의 두 상황에서 처음 뺀 당구공과 두번째로 뺀 당구공의 색이 같음을 알 수 있으므로 당구공 n + 1 개가 들어있는 임의의 집합은 색이 같은 것만을 포함함

      생각한 답: 위의 두 상황에서 처음 뺀 당구공과 두번째로 뺀 당구공의 색이 같음을 알 수 없음

      답: 위의 두 상황에서 처음 뺀 당구공과 두번째로 뺀 당구공의 색이 같음을 증명할 수 없음

      풀이

      • 대부분의 사람들이 P(n) 이 참이라고 가정할 수 없다고 반론함 P(n) 이 진짜 참인지 모르는데 왜 사실이라고 가정하냐는 내용
      • 수학적 귀납법에서 필요한 것은 P(n) → P(n + 1) 이 참임을 보이는 것 뿐이므로 P(n) 이 정말로 참일 필요는 없음 p → q 에서 p → q 이다 라는 것을 증명한 것임 즉, 앞 쪽이 거짓이면 전체는 항상 참이됨 (앞이 정말로 사실인지 확인할 필요가 없음) 앞 쪽이 정말로 사실인지 알아야 증명할 수 있다고 하면, 수학적 귀납법에서 증명하려고 하는 목표가 P(n) 이 사실이라는 것인데 그러면 증명 자체가 의미가 없어짐 (증명하려는 것이 이미 사실이라고 밝혀짐)
      • 위 증명에서 실제로 잘못된 것은 다음 부분 위의 두 상황에서 처음 뺀 당구공과 두번째로 뺀 당구공의 색이 같음을 알 수 있으므로 ...
      • 처음 뺀 당구공과 두번째로 뺀 당구공의 색이 같다는 것은 공통 부분이 있다는 것인데, 실제로 n = 1 인 경우, 즉 n + 1 = 2 인 경우 공통 부분이 없음 n = 1 인 경우, 당구공 n + 1 개가 들어있는 임의의 집합에서 증명을 위해 1개를 꺼냈을 경우 다른 하나는 혼자 있기 때문에 무조건 참이 됨 마찬가지로 꺼낸 공을 다시 넣고 다른 공을 꺼내도, 남은 공이 혼자 있기 때문에 참이되는 것임 즉, n + 1 = 2 인 경우 두 당구공의 색이 같다는 것이 아무데서도 도출이 되지 않음
      • step 이라는 과정 중에, n = 1 인 경우 제대로 증명이 성립되지 않음
      • 이 증명에서 착각을 불러일으키는 것은 n 이 상당히 크다고 가정하는 것임

    • Infinitely Many Prime Numbers
      • 제대로 된 증명이지만 반론이 나오는 경우가 있음

      문제: Prime Number 의 개수는 무한히 많다는 다음 증명은 옳은가?

      • Prime Number 의 개수가 유한한 k 개라고 가정
      • 모든 Prime Number 를 다 곱하고 1 을 더한 수를 n 이라고 하자
      • 이 수 n 은 어떤 Prime 으로 나누어도 나머지가 1 이다
      • 그런데 n 은 어떤 Prime 보다도 크므로 합성수이다
      • 합성수이지만 어떤 Prime 으로도 나누어지지 않으므로 모순 발생

      풀이

      • 이 증명에 대한 반론으로 몇 개의 Prime 이 더 존재하면 되는 것이 아니냐는 주장이 자주 있음 무한하다는 것이 증명된 게 아니라 k 개보다 많다는 것만 증명이 되었다는 뜻으로, n 보다 더 큰 Prime Number 가 존재한다면 모순이 해소되는 것이고 무한히 많다는 것은 아니라는 주장
      • 위 증명은 "Prime Number 가 k 개 이면 모순이 발생", 즉 "Prime Number 가 k 개" → "항상 거짓", 이 명제가 항상 참임을 확인한 것 뒷쪽이 거짓이므로 전체가 참이 되려면 앞쪽이 거짓일 수 밖에 없음
      • 즉, "Prime Number 가 k 개" 라는 명제가 항상 거짓일 수 밖에 없다!
      • Prime Number 의 개수가 무한하다는 것을 증명하기 위해 반대로 Prime Number 의 개수를 유한한 k 개 라고 가정함 모든 Prime Number 는 P1, P2, ... , Pk 라고 볼 수 있음
      • 모든 Prime Number 를 다 곱하고 1 을 더한 수 (n) 는 P1 X P2 X ... X Pk + 1 임 즉, n 은 어떤 Prime Number 보다 큰 수이므로 가장 큰 Prime Number 보다 큰 수임
      • n 은 Prime 이 아니므로 많이 알려진 사실에 의해 Prime Number 로 소인수분해가 되어야 함 어떠한 Prime Number 든지 n 을 나누었을 때 나누어 떨어져야 함
      • n 은 마지막에 1을 더했기 때문에 소인수분해가 불가능함 (나누어 떨어지지 않음) 1 때문에 모순이 발생하고, Prime Number 가 무한히 많다는 것을 증명할 수 있음

    • 수학적 귀납법과 증명의 수준
      • 컴퓨터 프로그래밍에서 사용되는 알고리즘은 수학적 귀납법으로 증명이 되는 경우가 굉장히 많음
      • 수학적 귀납법의 기본형: P(1) 이 참이고, P(n) → P(n + 1) 이 참이면 P(n) 은 모든 자연수 n 에 대해서 참이다.
      • 수학적 귀납법의 강한 형태: P(1) 이 참이고, P(1) ^ P(2) ^ ... ^ P(n) → P(n + 1) 이 참이면 P(n) 은 모든 자연수 n 에 대해서 참이다. P(1) 이 참이고, P(1), P(2), ... , P(n) 까지가 모두 참일 경우 P(n + 1) 이 참이다 라는 것을 증명해도 된다는 내용임 가정할 것이 많으므로 비교적 증명하기가 쉬움
      • 다음 함수가 1 부터 x 까지의 합을 계산함을 증명해 보자
        int sum(int x) {
	if (x <= 0) return 0;
	return x + sum(x - 1);
}
